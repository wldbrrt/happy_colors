# Happy Colors Playable Ad

Разработан с использованием движка https://github.com/lojkofob/eeditor

Тестовый playable ad, демонстрирующий отрисовку SVG-изображений без использования растровых текстур.

Проект показывает полный пайплайн:
- парсинг SVG в бинарный формат;
- загрузку данных в рантайме;
- создание mesh-буферов;
- отрисовку изображения;
- интерактивную раскраску;
- выбор цветов;
- анимацию и изменение расположения элементов.

---

## Project Structure

### SVG preprocessing

**`get_svg_bin_data.js`**

Преобразует SVG-файл в компактный бинарный формат, пригодный для использования в рантайме.

Основные задачи:
- парсинг SVG;
- извлечение контуров;
- сериализация данных;

---

### Assets & Layouts

**`happy_colour_001/`**

Содержит игровые ассеты:
- UI layouts;
- изображения;
- конфигурации сцены;

---

### Runtime mesh generation

**`src/binDataToMeshBuffers.js`**

После загрузки бинарных данных выполняет их преобразование в GPU-буферы.

Основные задачи:
- чтение бинарного формата;
- восстановление геометрии;
- создание vertex/index buffers;
- подготовка данных для рендера.

---

### Image rendering

**`src/behaviours/happy_colour.js`**

Основная логика раскрашивания изображения.

Отвечает за:
- обновление цветов;
- обработку пользовательского ввода;
- проверку выбранных областей;
- обновление состояния изображения.

---

### Color panel

**`src/behaviours/colors_panel.js`**

Реализует интерфейс выбора цветов.

Функциональность:
- отображение доступных цветов;
- обработка выбора пользователем;
- синхронизация выбранного цвета с игровой логикой.

---

## Rendering Pipeline

```text
SVG
 │
 ▼
get_svg_bin_data.js
 │
 ▼
Binary data
 │
 ▼
binDataToMeshBuffers.js
 │
 ▼
Vertex / Index Buffers
 │
 ▼
happy_colour.js
 │
 ▼
GPU Rendering
```
Пайплайн позволяет:
- уменьшить размер ассетов;
- отказаться от хранения готовых изображений;
- генерировать mesh в рантайме;
- динамически изменять цвета отдельных областей изображения.
